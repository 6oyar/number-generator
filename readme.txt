Тестовая работа свалилась нежданчиком, как снег на голову, немного не удобно сейчас её выполнять, но в целом сделал и есть несколько косяков:

 - Рандом не учитывает ранее выданные номера, всё-таки хранить их надо в большом количестве, и, чтобы не завалить сервер по памяти, 
надо время, чтобы подумать как всё это хранить, сегодня и в ближайшие выходные не могу уделить столько времени. 
Вообще навскидку в голову приходит хранить уже созданные номера в TreeSet, чтобы быстро искать по уже выданным. 
Для рандома при старте приложухи завести ArrayList, где проинициализировать номера от первого до последнего - 
это для того чтобы можно было быстро рандомить по индексу. 
Выбранный по рандому номер из ArrayList'а удаляется и переходит в TreeSet уже выданных, 
но я навскидку не могу сказать, насколько это оптимально по памяти (наверное можно придумать что-то по-опитмальней).

 - В случае падения сервака не сохраняется состояние выданных/невыданных номеров, но при старте можно задать начальный номер в файле config.txt. 
 Было бы больше времени, сделал бы счерез SQLite. При каждом вызове GET-запроса заливал бы в базу текущий номер. И не было бы страшно падение сервера.
